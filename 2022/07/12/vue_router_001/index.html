在vue-cli中使用vue-router

​ 新版本的vue-cli模板中自带vue-router，以下从模板中不带vue-router的情况开始说起

  • 安装 vue-router：

    npm i --save vue-router@X

    X 代表项目中使用 vue 的版本，如果是 vue2，则 X 只能以 2 开头，如

    npm i --save vue-router@2

  • 配置路由

    • 在 src 文件夹下创建 router 文件夹

    • 在 router 文件夹中创建 index.js 文件，文件中代码示例如下

      import Vue from "vue";  // 引入 Vue
import VueRouter from 'vue-router'; // 引入 vue-router

// 引入路由组件
import Home from 'views/Home';
import Login from 'views/Login';

// Vue 全局使用 Router
Vue.use(VueRouter);

// 配置路由
export default new VueRouter({
  routes: [
    {
      path: '/home',    //路由访问路径
      component: Home,  // 对应组件
    },
    {
      path: '/login',
      component: Login,
    },
    // 路由重定向，重定向根页面 '/' 至想要的路由组件
    {
      path: '/',
      redirect: '/home'
    }
  ]
});

  • 引入路由

    在配置完路由信息后，要想使用路由，还需要在 vue 中引入配置好的路由，在 src/main.js 中写入如下代码：

    import Vue from 'vue'
import App from './App.vue'

// 引入路由配置信息
import router from '@/router/index'

Vue.config.productionTip = false

new Vue({
  render: h => h(App),
  // 将路由注册至 vue
  router,
}).$mount('#app')

    路由注册后，所有的 vue 组件中都可以访问到 $route、$router 属性：

    • $route：某个路由组件的信息（路径、query、params等）
    • $router：整个路由器的信息（历史路径、编程导航等）

vue-router的基本使用

  • 路由的跳转

    • 声明式导航：

      <router-link to="/search">跳转到search</router-link>

      to 属性是将要跳转的路由路径

    • 编程式导航

      this.$router.push('/search');
this.$router.replace('/search')
...

      • 参数传递

        • params ：

          • 字符串形式下

            • 需要在路由配置中提前占位
            • 参数会显示在地址栏中

          • 对象形式下

            • 无需提前占位，如果占位，参数会显示在地址栏中。

            • 如果需要传递 params 参数，需要在路由配置中为该路由起一个别名 name，相应地，函数中使用 name 而不是 path。即：对象形式下，path 和 params 不能共存，否则会出现错误。编程式导航 | Vue Router (vuejs.org)

          // 占位配置
// router/index.js
{
    path: '/search/:searchWord', 
    component: Search,
},
// 传参
// views/Header/index.vue/script
jumpSearch(){
    // 普通字符串形式
    	this.$router.push('/search/' + this.searchWord);
    // 模板字符串形式
    	this.$router.push(`/search/${this.searchWord}`)
	// 对象形式
        this.$router.push({
            name: 'search',
            // 使用 name 时：
                // 如果路由配置中了 params 参数占位字符且字符后没有 '?'，则配置的参数必须传递，否则会报错
                // 添加 '?' 则代表该参数可传可不传
            params: {
                searchWord: this,searchWord,
            }
        })
    	// 错误写法
        this.$router.push({
            path: '/search',
            params: {
                searchWord: this,searchWord,
            }
        })
},
// 取参
this.$route.params.searchWord

        • query：

          • 无需提前占位
          • 会显示在地址栏，形式和get请求一样
          // views/Header/index.vue/script
jumpSearch(){
    // 普通字符串形式
    this.$router.push('/search?searchWord=' + this.searchWord);
    // 模板字符串形式
    this.$router.push(`/search?searchWord=${this.searchWord}`)
	// 对象形式
    this.$router.push({
        name: 'search',
        query: {
            searchWord: this,searchWord,
        }
    })
    this.$router.push({
        path: '/search',
        query: {
            searchWord: this,searchWord,
        }
    })
},
// 取参
this.$route.query.searchWord

  • 路由元信息的使用

    {
    path: '/register',
    component: Register,
    meta: {
        hiddenFooter: true,
    },
},

    这里的 meta 用来定义针对该路由的一些配置信息，可以在对应组件中通过以下方式访问

    computed: {
    hiddenFooter(){
      return this.$route.meta.hiddenFooter
    },
  },
    <Footer v-show="!hiddenFooter"></Footer>

tips

  • 对象传参形式下，params 是否可以传空字符串？

    可以，但需写成如下方式，否则地址栏路径会出错

     this.$router.push({
     name: 'search',
     params: {
         searchWord: '' || undefined,
     }
 })
 
+ 路由传参可以使用 props 接收吗？

可以，有三种配置方法：

```javascript
// router/index.js
{
    path: '/search/:searchWord', 
    name: 'search',
    component: Search,
   	// 1.路由携带参数可以在 props 中访问到
    props: true,
    // 2.传递给组件的 props 一些自定义参数
    props: {a: 1, b: 2},
    // 3.上面两种的结合
    props: ($route) => ({
        searchWord: $route.params.searchWord,
        haha: $route.query.haha,
        a: 1,
        b: 2,
    })
},

  • 解决编程式导航 参数不变多次跳转路由报错的问题 bilibili尚硅谷VUE项目实战，前端项目-尚品汇

    // router/index.js

// 1. 直接捕获错误
const originalPush = VueRouter.prototype.push;
VueRouter.prototype.push = function push(location) {
  return originalPush.call(this, location).catch(err => err)
};

// 2. 检查使用 push 或 replace 时，是否传入了 promise 处理函数，如果没有，就加上。
const originalReplace = VueRouter.prototype.replace;
VueRouter.prototype.replace = function replace(location, resolve, reject) {
  if(resolve || reject){
    originalReplace.call(this, location, resolve, reject);
  } else {
    originalReplace.call(this, location, () => {}, () => {})
  }
};
